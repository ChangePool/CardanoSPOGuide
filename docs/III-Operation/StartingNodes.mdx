Now, the Cardano Node instances in your stake pool configuration are ready to start. Initially when you start Cardano Node, the instance begins synchronizing a local copy of the Cardano blockchain ledger with the network.

<p class="pi">To start each instance of Cardano Node in your stake pool configuration:</p>

<ol>
  <li>On a computer hosting a relay node, using a terminal window type the command <code>sudo systemctl start cardano-node</code><center><strong>OR</strong></center>Reboot the computer.</li>
  <li>Repeat step 1 on all computers comprising your stake pool configuration.</li>
</ol>

To monitor the progress of a Cardano Node instance synchronizing a local copy of the the blockchain with the network, follow entries in the log.

<p class="pi">To follow the log for an instance of Cardano Node:</p>

<ul class="ss">
  <li>Using a terminal window on the computer hosting the instance of Cardano Node that you want to monitor, type <code>journalctl --follow --unit cardano-node</code></li>
</ul>

Following a timestamp, each entry in the Cardano Node log begins with the name of a trace enclosed in square brackets (`[` and `]`), followed by the logging data generated by the trace.

Traces use a hierarchical structure. The naming convention for traces separates each tier in the hierarchy using a period (`.`) character. Each trace logs a different aspect of Cardano Node operation.

## Managing Services

To help administer an instance of Cardano Node running as a `systemd` service, use the following commands:
```bash
# Display the status of the service
sudo systemctl status cardano-node
# Restart the service
sudo systemctl reload-or-restart cardano-node
# Stop the service
sudo systemctl stop cardano-node
```

For more details on using the `systemctl` utility to manage services, see the [manual page](https://man7.org/linux/man-pages/man1/systemctl.1.html).

## Monitoring the Cardano Node Log

Regular monitoring of entries in the log is essential for determining how an instance of Cardano Node is running.

**To display and filter the Cardano Node log:**
```bash
# Show the most recent log entries, and continuously print new entries as they are appended to the log
journalctl --unit=cardano-node --follow
# Show entries appended to the log since midnight the day before today
journalctl --unit=cardano-node --since=yesterday
# Show entries appended to the log on the current day
journalctl --unit=cardano-node --since=today
# Show entries appended to the log on July 29, 2025 between midnight and noon
journalctl --unit=cardano-node --since='2025-07-29 00:00:00' --until='2025-07-29 12:00:00'
```

For more details on using the `journalctl` utility, see the [manual page](https://man7.org/linux/man-pages/man1/journalctl.1.html).

If you prefer, you can also use a different log viewer—such as [The Logfile Navigator](https://lnav.org/) or [glogg](https://glogg.bonnefon.org/) for example—to review the log more easily.

**To review the Cardano Node log using The Logfile Navigator:**

1\. To install The Logfile Navigator, using a terminal window type:
```bash
sudo apt-get install lnav
```

2\. To follow Cardano Node log entries using The Logfile Navigator, type:
```bash
journalctl --unit=cardano-node --follow | lnav
```

**To review the Cardano Node log using `glogg`:**

1\. To install `glogg` type:
```bash
sudo apt-get install glogg
```

2\. To display log entries that Cardano Node generated within a date range using `glogg` type:
```bash
journalctl --unit=cardano-node --since='2025-07-29 00:00:00' --until='2025-07-29 12:00:00' | glogg
```
